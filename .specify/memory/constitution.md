# FavToSkill 项目章程

## 核心原则

### I. 产品定位
将用户的浏览器收藏夹转化为可对话、可互动的知识技能体系，帮助用户沉淀和激活已收藏但未消化的内容。

### II. 设计语言规范

#### 视觉风格：柔和温暖的极简主义（Soft Minimal）

**配色体系**
- 主背景：浅蓝绿渐变（`#D8EEF0` → `#C5E8EA`），营造平静氛围
- 主色调：柔绿（`#6DBF9E` 系列），代表成长与冷静
- 卡片背景：纯白（`#FFFFFF`），高对比度突出内容
- 文字主色：深炭黑（`#1A1A1A`），辅助文字浅灰（`#888888`）
- 强调色：湖蓝（`#4BBFD4`）用于图标和交互元素

**排版规范**
- 标题字体：衬线体（Serif），大号加粗，传递可信度与沉稳感
- 正文字体：无衬线体（Sans-serif），清晰易读
- 文字层级：大标题 → 副标题 → 标签，层次分明，留白充足
- 标签/Tag：圆角胶囊形，细描边，字号小，用于分类展示

**圆角与形态**
- 卡片圆角：16px–24px，大圆角营造亲和感
- 按钮：全圆角（pill shape），主按钮深色实心，次按钮文字链
- 图标容器：圆形背景，填充柔和色彩
- 有机形态（blob shape）：装饰性插画使用不规则圆润形状

**插画与图标**
- 吉祥物/角色：卡通云朵、动物等圆润可爱造型，增加情感连接
- 图标风格：线性 + 少量填充，简洁不繁琐
- 装饰图形：使用半透明叠层、渐变模糊营造景深

**交互与动效原则**
- 卡片可滑动/堆叠展示，增加探索感
- 底部 Tab 栏：深色药丸形容器，图标高亮清晰
- 情绪/状态选择：水平滑动选项，选中项高亮区分
- 整体动效：轻柔、平滑，避免突兀跳转

**空间与布局**
- 大量留白，内容不拥挤
- 单列为主，重点内容居中对齐
- 卡片作为主要信息容器，内容块感强

### III. 技术栈约定
- 前端：React + CSS Modules / Tailwind CSS
- 运行环境：Web（桌面优先，兼顾移动端响应式）
- 数据：Mock 数据驱动，演示优先

### IV. Demo 优先原则
当前阶段以产品概念验证为目标，优先实现视觉表达和用户流程演示，不过度追求工程完整性。

### V. 简洁性
从最简可行方案出发，YAGNI 原则，每次迭代聚焦单一核心场景。

### VI. 响应式设计与多端素材适配规范

#### 断点定义
- **移动端（Mobile）**: 320px - 767px
- **平板端（Tablet）**: 768px - 1023px
- **PC端（Desktop）**: 1024px 及以上

#### 素材规格要求

**背景图/Banner 素材**
- PC端：推荐尺寸 1920×1080px 或 2560×1440px，横向16:9比例
- 移动端：推荐尺寸 750×1334px 或 1080×1920px，竖向9:16比例
- 格式要求：WebP 优先（带 PNG/JPG 降级），文件大小 < 500KB

**卡片/组件背景**
- PC端：1200×600px 或对应宽高比
- 移动端：750×600px 或对应宽高比
- 必须提供 @2x / @3x 高清版本（移动端）

**头像/图标素材**
- 统一尺寸：512×512px（方形），自适应容器大小
- 使用 SVG 格式优先，PNG 次之

#### 开发实践规范

**1. 素材上传提示**
在需要上传背景、Banner 等大型素材时，系统应：
- 检测当前视口宽度，明确提示用户当前设备类型
- 提供清晰的尺寸要求说明：
  ```
  请上传适配当前设备的素材：
  - 移动端：750×1334px（9:16竖图）
  - PC端：1920×1080px（16:9横图）
  ```
- 支持同时上传多端素材，自动根据设备加载对应版本

**2. 响应式图片加载**
```html
<picture>
  <source media="(min-width: 1024px)" srcset="bg-desktop.webp" />
  <source media="(min-width: 768px)" srcset="bg-tablet.webp" />
  <img src="bg-mobile.webp" alt="Background" />
</picture>
```

**3. CSS 媒体查询规范**
```css
/* Mobile First 策略 */
.component {
  /* 移动端样式 (default) */
}

@media (min-width: 768px) {
  /* 平板端样式 */
}

@media (min-width: 1024px) {
  /* PC端样式 */
}
```

**4. 布局适配原则**
- 移动端：单列布局，卡片堆叠，触摸优化（最小点击区域 44×44px）
- PC端：多列网格（2-3列），鼠标悬停效果，导航侧边栏
- 图片容器使用 `object-fit: cover` 保持比例裁剪
- 避免固定像素宽度，使用百分比/视口单位（vw/vh）

**5. 性能优化要求**
- 使用 `loading="lazy"` 延迟加载非首屏图片
- 移动端默认加载低分辨率版本
- 使用 CDN 分发静态资源
- 利用浏览器缓存（Cache-Control 设置合理过期时间）

#### 验收标准
所有页面必须在以下设备/浏览器通过测试：
- 移动端：iPhone 12/13/14、Android（Chrome）
- PC端：Chrome、Safari、Firefox（最新版本）
- 屏幕尺寸：375px、768px、1440px、1920px

## 设计参考来源
- 参考应用：Reflections（心理健康社区 App）
- 风格关键词：Soft UI、Calm Design、Friendly Minimal、Organic Shape

## 治理
本章程优先于所有其他实践。设计决策须符合"柔和温暖极简主义"风格规范，新增 UI 组件前需对照本章程检查一致性。
